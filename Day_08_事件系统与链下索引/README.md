## 🎯 为什么重要

事件系统是区块链与外部世界通信的桥梁：
- **链下监听**：前端和后端可以监听事件并做出响应
- **数据索引**：索引器通过事件构建高效的查询系统
- **审计追踪**：事件提供了完整的操作历史记录
- 在 DeFi 开发中，事件用于通知交易、价格变化、清算等关键操作

## ⚠️ 常见陷阱

1. **事件不能在链上读取**：只能被链下监听
2. **EventHandle 必须存储在资源中**：不能作为函数参数传递
3. **事件顺序很重要**：按发射顺序依次编号
4. **事件数据要精简**：避免存储大量数据
