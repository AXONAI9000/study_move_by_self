# Day 05 - 模块系统与可见性

## 学习目标
- 掌握模块（Module）的定义与组织
- 理解模块的导入与使用
- 学习可见性控制规则
- 了解friend声明与模块间协作

## 核心内容
1. 模块定义：module关键字、命名规范
2. 地址与模块路径
3. 导入语句：use关键字、as别名
4. 可见性修饰符：public、public(friend)、internal
5. friend模块声明
6. 标准库模块的使用

## 为什么重要？
模块系统是代码组织的基础，良好的模块设计能提高代码的可维护性和安全性，这些知识将在后续学习泛型和高级特性时频繁使用。

## 衔接下一阶段
掌握了前5天的基础知识后，你将具备学习Day_06泛型编程的先决条件，能够理解类型参数化和能力约束系统。
